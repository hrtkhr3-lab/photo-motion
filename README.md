# 写真モーション（photo-motion）

🔗 **公開サイト: https://hrtkhr3-lab.github.io/photo-motion/**

写真を1枚アップロードし、「お題（どう動かしたいか）」をテキストで入力すると、その写真に
**カメラワーク・エフェクト・色変化**を重ねて“動いて見える”アニメーションを作り、**動画（webm）として保存**できる Web アプリです。

- ✅ **ランニングコスト 0円** — 有料API・課金は一切なし
- ✅ **APIキー不要** — 秘密情報がないので無料の静的ホスティングにそのままデプロイ可能
- ✅ すべて**ブラウザ内**で処理（サーバー不要）

## ⚠️ 方式について（期待値合わせ）

本アプリは**写真の被写体そのものを生成AIで動かすものではありません**。
1枚の静止画のまま、その上に Ken Burns（ズーム/パン）・パーティクル（雨/雪/光/キラキラ）・
カラーグレード（夕暮れ/夜など）といった演出を重ねて“動いて見える”動画を作ります。

- **できること**：ゆっくりズーム/パン、雨・雪・光・キラキラの重ね合わせ、夕暮れ等の色・明るさ変化、そよ風のような微細な揺れ
- **できないこと**：髪が物理的になびく／人物が動き出す／写真に無い要素が生成される——といった“本物の生成AI動画”

## 必要なもの

- Node.js（LTS 推奨）/ npm
- モダンブラウザ（**Chrome 推奨**。動画保存は `MediaRecorder` による webm 出力）

## 起動（ローカル開発）

```bash
npm install
npm run dev
```

表示された `http://localhost:5173` をブラウザで開きます。

## ビルド

```bash
npm run build      # dist/ に静的ファイルを出力
npm run preview    # ビルド成果物をローカル確認
```

## デプロイ（無料の静的ホスティング）

`dist/` を配信するだけです。`vite.config.ts` で `base: './'`（相対パス）にしてあるため、
サブパス配信でもそのまま動きます。

- **GitHub Pages**：`dist/` の中身を `gh-pages` ブランチへ push、または Actions で配信
- **Cloudflare Pages / Vercel 無料枠**：ビルドコマンド `npm run build`、出力ディレクトリ `dist`

## 使い方

1. 写真をドラッグ＆ドロップ（またはクリックで選択）
2. お題を入力（例：「夕暮れの中をゆっくり寄っていく」「雨が降る幻想的な雰囲気」）。プリセットボタンでも追加可能
3. アスペクト比（16:9 / 9:16 / 1:1）と長さ（4〜8秒）を選ぶ
4. **「プレビュー生成」**で Canvas 上のアニメーションを確認
5. **「動画として保存」**で webm をダウンロード

## アーキテクチャ

お題の解釈とレンダリングを **演出仕様(JSON)** で分離しています。解釈器を差し替えても
レンダラ側は無変更で動きます（`src/types.ts` の `Spec` が境界）。

```
src/
├─ types.ts            # 演出仕様(JSON)の型（解釈器⇔レンダラの境界）
├─ interpret/
│  └─ rules.ts         # お題→演出仕様：ルールベース解釈器（0円・オフライン・既定）
├─ render/
│  ├─ renderer.ts      # メインレンダラ（Ken Burns＋揺れ→色→エフェクト）
│  ├─ effects.ts       # 雨/雪/光線/キラキラ/ほこり/玉ボケ
│  ├─ colorgrade.ts    # ムード・明るさカーブ・ビネット
│  ├─ easing.ts        # イージング
│  └─ recorder.ts      # MediaRecorder で webm 録画
├─ components/         # Uploader / PromptInput / Presets / Player
└─ App.tsx
```

## 奥行きパララックス（Step5・実装済み）

「奥行きパララックス」にチェックを入れると、**transformers.js の単眼デプス推定（Depth-Anything V2 small）を
ブラウザ内で実行**し、人物（前景）と背景を分離します。カメラ移動時に前景と背景を別々に動かすことで、
**人物が背景から浮き出て立体的に動いて見える**ようになります。

- モデルは HuggingFace の CDN から取得（**無料・APIキー不要**、初回のみ数十MB）
- WebGPU があれば高速、なければ WASM(CPU) で実行（やや遅い）
- 失敗・未対応の環境では自動でパララックスをオフにし、通常の Ken Burns ＋エフェクトで成立

> ⚠️ これは“奥行きのある見かけの動き”です。髪が物理的になびく・表情が変わるといった
> 本物の生成AI動画ではありません（それには有料の image-to-video API が必要）。
